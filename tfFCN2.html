        TensorFlow代码实现FCN框架的语义分割（二）：使用VGG预训练网络_获取中间层输出

        • 博主： 52txr

        在本次训练中，使用VGG预训练神经网络来进行语义分割。在前面一节『TensorFlow代码实现FCN框架的语义分割（一）：Oxford-IIIT Pet图片的读取集dataset创建』中已经完成了训练集以及测试集的制作。在本节进行网络的创建。

        关于预训练神经网络，可以看看之前的文章：

        • 预训练网络(迁移学习)基础知识_VGG模型简介及缺点
        • 迁移学习_小数据集获取极高的正确率之预训练网络的微调操作
        • 迁移学习代码实现：使用VGG16对2000猫狗数据集分类训练

        在预训练网络中创建子模型

        为什么要创建子模型？因为FCN框架需要调用前面的layer。

        在制作数据集的时候，对所有的图片都处理为(224,224,3)的大小。这里只需要使用这个网络提取特征，而不用它进行分类，所以include_top = False，在之前的预训练网络中皆有介绍。

        conv_base = tf.keras.applications.VGG16(weights='imagenet', 
                                        input_shape=(224,224,3),
                                        include_top = False)

        查看网络模型的结构：

        在前面说到，FNC的框架具有跳级结构（『图像语义分割网络结构FCN架构简介（二）_跳级结构skip』）。也就是后面输出的要和前面卷积提取特征部分进行联合（如下图所示的FCN框架）。那么，输出的和前面的卷积部分的哪一个layer进行联合呢？

        在最后的block5_pool进行上采样，依次和block5_conv3、block4_conv3、block3_conv3、block2_conv2进行反卷积，得到恢复的图片。在tensorflow中，可以使用get_layer()这个函数获得所有的层。还可以使用out属性获得它的输出。如代码所示：

        下面就可以尝试进行创建子模型了，创建的sub_mode实际上会继承权重。

        sub_model = tf.keras.models.Model(inputs=conv_base.input,
                                  outputs=conv_base.get_layer('block5_conv3').output
                                 )

        获取模型中间层的输出

        在上面简单实现了模型的截断，获得sub_model。现在要对block5_conv3、block4_conv3、block3_conv3进行截断。其实这样就是一个多输出。这里使用列表的形式进行获取。

        layer_names = ['block5_conv3',
               'block4_conv3',
               'block3_conv3',
               'block5_pool']

layer_output = [conv_base.get_layer(layer_name).output for layer_name in layer_names]

multi_out_model = tf.keras.models.Model(inputs=conv_base.input,
                                  outputs=layer_output
                                 )

multi_out_model.trainable = False
